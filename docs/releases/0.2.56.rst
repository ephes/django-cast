0.2.56 (unreleased)
-------------------

- Require Wagtail 7.4, ``django-tasks`` 0.12, and ``django-tasks-db`` 0.12 for
  Voxhelm transcript generation, and document the ``cast_transcripts`` database
  worker setup for Wagtail admin transcript completion.
- Limit the supported Wagtail range to ``>=7.4,<7.5`` and align the default
  tox matrix with that supported range by removing Wagtail 6 and Django 4.2
  environments.
- Drop support for Django 4.2; Django 5.2 or higher is now required because
  Wagtail 7.4 no longer supports Django 4.2.
- Replace any hand-managed ``django_tasks.backends.database`` entry in
  ``INSTALLED_APPS`` with ``django_tasks`` and ``django_tasks_db``.
- Remove any legacy ``ENQUEUE_ON_COMMIT`` keys from ``TASKS`` settings; the
  current ``django-tasks`` configuration no longer uses that option.
- Require ``modelsearch>=1.3,<1.4`` so full-text search remains compatible
  with ``django-tasks`` 0.12 while staying inside Wagtail's supported
  ``modelsearch`` range.
- Replace the raw ``cloc`` plus ``sloccount`` ``just loc`` recipe with the
  shared ``slopscope`` line-count workflow via a local editable checkout. The
  command keeps ``cloc`` semantics when ``cloc`` is available and now reports
  language, source/test, area, and directory summaries instead of the old
  ``sloccount`` folder-only appendix.
